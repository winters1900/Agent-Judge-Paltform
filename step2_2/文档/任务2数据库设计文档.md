# 任务2 数据库设计文档

## 1. 文档概述

### 1.1 文档名称
Agent 应用评估平台数据库设计文档

### 1.2 设计目标
本文档用于定义任务2评估平台的数据库结构，支撑评测任务管理、评测执行、数据集管理、指标配置、过程轨迹记录、结果分析与报告导出等功能。

### 1.3 设计原则
- 结构清晰，满足前后端分离和异步评测场景
- 任务元数据、运行数据、样本数据、指标数据分表存储
- 过程轨迹与结果汇总分离，便于查询优化
- 支持扩展自定义指标和多种评估方法
- 关键字段建立索引，保证列表查询和详情查询性能

---

## 2. 数据库选型

### 2.1 主数据库
- **MySQL 8.x**

### 2.2 选型理由
- 支持事务，适合任务状态流转与结果持久化
- JSON 字段可存储灵活配置
- 索引与查询能力成熟，适合管理型平台
- 与主流后端框架兼容性好，部署简单

### 2.3 辅助存储建议
- **Redis**：用于任务队列、运行状态缓存、实时进度缓存
- **对象存储/文件系统**：用于评测报告导出文件、数据集文件、日志文件

## 3. 数据表设计

## 3.1 评测任务表 `evaluation_task`
用于记录评测任务的基本信息、配置和状态。

#### 任务状态统一约定
- `draft`：草稿
- `pending`：待执行
- `running`：执行中
- `succeeded`：成功
- `failed`：失败
- `cancelled`：已取消
- `archived`：已归档

### 字段设计
| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | bigint | PK, AI | 主键 |
| task_code | varchar(64) | UNIQUE, NOT NULL | 任务编码 |
| name | varchar(128) | NOT NULL | 任务名称 |
| description | text | NULL | 任务描述 |
| agent_id | varchar(64) | NOT NULL | 被测 Agent 标识 |
| dataset_id | bigint | NOT NULL | 数据集 ID |
| status | varchar(32) | NOT NULL | 任务状态 |
| metric_config | json | NOT NULL | 指标配置 |
| evaluation_method_config | json | NOT NULL | 评估方法配置 |
| run_config | json | NOT NULL | 运行配置 |
| created_by | bigint | NULL | 创建人 |
| updated_by | bigint | NULL | 更新人 |
| created_at | datetime | NOT NULL | 创建时间 |
| updated_at | datetime | NOT NULL | 更新时间 |
| deleted_at | datetime | NULL | 软删除时间 |

### 索引建议
- `idx_task_status(status)`
- `idx_task_agent_id(agent_id)`
- `idx_task_dataset_id(dataset_id)`
- `idx_task_created_at(created_at)`

## 4. 表关系说明

### 4.1 核心关系
- `evaluation_task` 1:N `evaluation_run`
- `dataset` 1:N `dataset_sample`
- `evaluation_run` 1:N `evaluation_sample_result`
- `evaluation_run` 1:N `evaluation_trace`
- `evaluation_run` 1:N `tool_call_log`
- `evaluation_run` 1:N `metric_result`
- `evaluation_run` 1:1/1:N `evaluation_report`
- `evaluation_method` 与 `evaluation_task` 通过配置关联
- `metric_definition` 与 `metric_result` 通过 `metric_id` 关联

### 4.2 逻辑说明
- 任务只保存配置和状态，不保存大量运行明细
- 运行表保存一次完整评测过程
- 样本结果表用于支撑单样本分析和问题定位
- 轨迹表与工具日志表用于过程可视化和异常排查
- 指标结果表用于统一汇总显示和对比分析

## 6. 性能与维护建议

### 6.1 分区或归档建议
- 运行明细较多时，可按月份或任务维度归档历史运行数据
- `evaluation_trace` 和 `tool_call_log` 数据量增长快，建议按运行时间分表或分区

### 6.2 缓存建议
- 任务列表、指标列表、方法列表可缓存到 Redis
- 运行进度可实时写入缓存并通过 WebSocket 推送

### 6.3 数据清理建议
- 取消或失败任务保留原始日志，便于问题排查
- 报告文件和导出结果设置生命周期管理
