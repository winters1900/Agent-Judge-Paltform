# 任务2需求文档

## 1. 项目概述

### 1.1 项目名称
Agent 应用评估平台

### 1.2 项目背景
本项目面向迭代二的任务二，目标是构建一个用于评估 Agent 应用的 Web 平台。平台需要支持评测任务的创建、配置、执行、监控、结果展示与对比分析。评测对象可以是本项目自研的 Agent 应用，也可以是其他符合接口规范的 Agent 服务。

### 1.3 项目目标
- 为 Agent 应用提供统一的评测入口
- 支持多种评估维度与评估方法
- 支持任务级、运行级、指标级的结果管理
- 支持过程数据留存与可视化
- 支持单次评测与多次评测对比分析

### 1.4 范围说明
本平台重点覆盖以下能力：
- 评测任务管理
- 评测执行管理
- 评测指标计算
- 评测结果展示
- 多任务对比分析
- 自定义评估指标扩展

---

## 2. 用户角色

### 2.1 平台使用者
- 创建评测任务
- 配置被测 Agent 版本、数据集与评估方法
- 启动或停止评测任务
- 查看评测状态、过程与结果
- 对比多个任务的评测表现

### 2.2 系统管理员
- 管理平台配置
- 管理数据集与评测模板
- 维护评测方法与指标插件
- 查看系统运行状态与日志

---

## 3. 核心业务流程

1. 用户创建评测任务
2. 用户选择被测 Agent、数据集、评估方法和指标
3. 系统生成评测实例并进入排队或运行状态
4. 系统调用被测 Agent 执行数据集中的测试样本
5. 系统记录输入、输出、过程轨迹和工具调用日志
6. 系统根据评估方法计算指标
7. 系统生成单次评测结果与汇总报告
8. 用户可查看结果详情并进行多任务对比

---

## 4. 功能需求

### 4.1 评测任务管理
#### 4.1.1 创建评测任务
系统应支持创建评测任务，并支持配置以下信息：
- 任务名称
- 任务描述
- 被测 Agent 版本
- 数据集选择
- 评估方法选择
- 评估维度选择
- 指标配置
- 运行参数（超时、并发数、重试策略等）

#### 4.1.2 查询评测任务
系统应支持按任务名称、状态、创建时间等条件查询评测任务。

#### 4.1.3 修改评测任务
系统应支持修改未开始或未完成的评测任务配置。

#### 4.1.4 删除评测任务
系统应支持删除未开始或已归档的评测任务。

#### 4.1.5 任务状态管理
评测任务状态统一定义为：
- draft
- pending
- running
- succeeded
- failed
- cancelled
- archived

### 4.2 评测执行管理
#### 4.2.1 评测启动
系统应支持手动启动评测任务。

#### 4.2.2 评测停止
系统应支持在任务运行中停止评测。

#### 4.2.3 评测过程记录
系统应记录每次评测执行的：
- 开始时间
- 结束时间
- 执行耗时
- 执行状态
- 错误信息
- 中间步骤轨迹
- 工具调用日志

#### 4.2.4 断点与异常处理
当被测 Agent 发生超时、接口异常、输出格式错误或工具调用失败时，系统应能够记录异常并给出可读的失败原因。对于可恢复场景，系统应支持重试；对于不可恢复场景，系统应保留完整失败记录。

### 4.3 评估方法管理
系统应支持多种评估方法，且这些方法可以重叠组合使用。

#### 4.3.1 按评估模式划分
- **面向结果**：只关注输入与输出，不强依赖中间过程
- **面向过程**：关注 Agent 的中间思考、工具调用与反思过程

#### 4.3.2 按评估方式划分
- **显式指标**：可直接计算的客观指标
  - 任务成功率
  - 响应时间
  - Token 消耗
  - 工具调用正确率
  - 重试次数
- **模糊指标**：依赖规则或 LLM-as-a-Judge 的主观评估
  - 推理质量
  - 输出完整性
  - 幻觉程度
  - 交互体验
  - 安全性判断

#### 4.3.3 按评估维度划分
- **效果维度**：结果是否有效、准确、完整
- **安全维度**：输出是否包含有害内容或违规内容
- **性能维度**：响应是否流畅，是否存在长时间停顿或卡顿

#### 4.3.4 评估引擎支持
平台应支持接入以下评估引擎或框架：
- Ragas
- 自定义规则引擎
- LLM-as-a-Judge 评估器
- 过程轨迹分析器

### 4.4 评测结果展示
#### 4.4.1 单次任务结果展示
系统应展示单个评测任务的：
- 基本信息
- 状态信息
- 指标汇总
- 详细样本结果
- 过程轨迹图或步骤日志
- 工具调用详情

#### 4.4.2 多任务对比分析
系统应支持多个评测任务之间的横向对比，至少支持：
- 任务成功率对比
- 平均响应时间对比
- 不同指标雷达图或柱状图展示
- 数据集维度下的结果对比

#### 4.4.3 结果导出
系统应支持将评测结果导出为 JSON、CSV 或 PDF 格式，并支持导出单次运行结果与对比分析结果。

### 4.5 数据集管理
系统应支持：
- 数据集创建、编辑、删除、查询
- 数据集样本导入与导出
- 样本字段定义与校验
- 按场景划分数据集版本

### 4.6 指标扩展能力
系统应支持：
- 自定义评估指标注册
- 指标组合策略配置
- 新评估方法插件化接入
- 指标结果可追踪与可复现

### 4.7 可视化与监控
系统应支持：
- 评测任务运行状态看板
- 实时进度展示
- WebSocket 推送任务状态变化
- 错误告警与异常提示
- 运行日志查看

---

## 5. 非功能需求

### 5.1 性能要求
- 支持并发执行多个评测任务
- 单次任务应能在合理时间内完成状态更新与结果展示
- 页面交互应保持流畅

### 5.2 可靠性要求
- 评测任务异常中断后应可恢复或保留完整失败记录
- 任务执行日志与评测结果应持久化存储

### 5.3 可扩展性要求
- 可新增评估指标
- 可新增被测 Agent 类型
- 可新增数据集与评测模板
- 可替换评估引擎

### 5.4 可维护性要求
- 前后端分离
- 接口设计清晰
- 数据模型规范
- 日志与错误定位方便

---

## 6. 推荐技术实现

### 6.1 前端
- Vue / React / 任意现代前端框架
- 图表库用于结果可视化
- WebSocket 用于实时状态更新

### 6.2 后端
- FastAPI / Django / Spring Boot / 任意后端框架
- 任务调度或异步队列用于执行评测
- RESTful API 提供任务管理与结果查询

### 6.3 评估能力
- Ragas 作为可复用评估库
- 自定义指标模块补充 Agent 特有评估需求

---

## 7. 主要页面需求

### 7.1 评测任务列表页
- 展示任务名称、状态、创建时间、评估对象、结果摘要
- 支持筛选、排序、分页

### 7.2 评测任务创建页
- 配置 Agent、数据集、评估方法、指标与运行参数

### 7.3 评测任务详情页
- 展示任务状态、运行日志、步骤轨迹、指标结果

### 7.4 结果对比页
- 选择多个任务进行对比
- 展示对比图表与差异分析

### 7.5 数据集管理页
- 维护评测样本与数据集版本

---

## 8. 与任务1的关系

任务2评估平台建议直接对接任务1的 Agent 应用。任务1应尽量提供以下数据，以便任务2进行评估：
- 输入参数
- 最终输出
- 中间步骤轨迹
- 工具调用日志
- 执行耗时
- 成功/失败状态
- 可选参考上下文或检索结果

这将使任务2可以同时支持：
- 结果型评估
- 过程型评估
- 显式指标评估
- 基于 Ragas 的指标评估

---

## 9. 验收标准

1. 可创建、编辑、删除、查询评测任务
2. 可启动并监控评测任务执行
3. 可查看单次评测结果与过程轨迹
4. 可对多个任务进行对比分析
5. 可接入 Ragas 或同类评估库
6. 可扩展自定义评估指标
7. 可展示评测任务状态与统计结果
