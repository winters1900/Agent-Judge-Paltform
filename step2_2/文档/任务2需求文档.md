# 任务2需求文档

## 1. 项目概述

### 1.1 项目名称
通用 Agent 评估平台

### 1.2 项目背景
本项目面向迭代二任务二，目标是搭建一个前后端分离的通用 Agent 评估平台，用于评估不同形态、不同接入方式、不同能力侧重的 Agent 应用。平台应支持结果型评估、过程型评估、显式指标评估、模糊指标评估以及可扩展的自定义评估能力，并能够兼容多种 Agent 场景，如对话类 Agent、工具调用类 Agent、工作流类 Agent、检索增强类 Agent、代码类 Agent 等。

平台中的“评测目标”是统一抽象，既可以是 HTTP Agent、CLI Agent、Workflow Agent，也可以是其它可适配的被测系统；AI Coding Agent 仅作为示例场景之一，用于展示平台在项目生成、代码修改、命令执行、过程记录与结果展示方面的能力，但 Coding 场景不应限制平台的通用性。

### 1.3 项目目标
- 为不同类型的 Agent 应用提供统一的评测入口
- 支持评测任务的创建、配置、执行、监控与管理
- 支持结果型评估与过程型评估
- 支持显式指标、模糊指标和自定义指标扩展
- 支持单次评测结果展示与多任务对比分析
- 支持多种评估方法组合使用
- 支持通用平台能力，并可通过示例场景展示具体 Agent 的评估流程
- 支持后续扩展更多 Agent 类型、数据集类型、指标类型和评估方法

### 1.4 项目范围
本平台重点覆盖以下能力：
- 评测任务管理
- 评测执行管理
- 数据集管理
- 评估方法与指标管理
- 结果展示与分析
- 多任务对比
- 自定义评估指标扩展
- 评测过程记录与可追溯分析

---

## 2. 用户角色

### 2.1 平台使用者
- 创建评测任务
- 配置被测 Agent、数据集与评估方法
- 启动、停止、取消或重试评测任务
- 查看评测状态、过程与结果
- 对比多个任务的评测表现

### 2.2 系统管理员
- 管理平台配置
- 管理数据集与评测模板
- 维护评估方法与指标插件
- 查看系统运行状态与日志
- 维护 Agent 接入配置与执行资源

---

## 3. 核心业务流程

1. 用户创建评测任务
2. 用户选择被测 Agent、数据集、评估方法和指标
3. 系统生成评测运行实例并进入排队或运行状态
4. 系统通过统一适配层调用被测 Agent 执行评测样本
5. 系统记录输入、输出、过程轨迹和工具调用日志
6. 系统根据评估方法计算样本级和运行级指标
7. 系统生成单次评测结果与汇总报告
8. 用户可查看结果详情并进行多任务对比

---

## 4. 功能需求

### 4.1 评测任务管理
#### 4.1.1 创建评测任务
系统应支持创建评测任务，并支持配置以下信息：
- 任务名称
- 任务描述
- 被测 Agent 配置
- 数据集选择
- 评估方法选择
- 评估维度选择
- 指标配置
- 运行参数（超时、并发数、重试策略等）
- 接入适配配置（如接口地址、调用凭证、协议类型等）

#### 4.1.2 查询评测任务
系统应支持按任务名称、状态、创建时间等条件查询评测任务。

#### 4.1.3 修改评测任务
系统应支持修改未开始或未完成的评测任务配置。

#### 4.1.4 删除评测任务
系统应支持删除未开始或已归档的评测任务。

#### 4.1.5 任务状态管理
评测任务状态统一定义为：
- `draft`
- `pending`
- `running`
- `succeeded`
- `failed`
- `cancelled`
- `archived`

### 4.2 评测执行管理
#### 4.2.1 评测启动
系统应支持手动启动评测任务，也应支持通过 API 或计划任务触发评测执行。

#### 4.2.2 评测停止、暂停与恢复
系统应支持在任务运行中停止、取消、暂停或恢复评测。

#### 4.2.3 评测重试与恢复
系统应支持对失败运行或失败样本进行重试，并在可恢复场景下支持断点恢复。

#### 4.2.4 评测过程记录
系统应记录每次评测执行的：
- 开始时间
- 结束时间
- 执行耗时
- 执行状态
- 错误信息
- 中间步骤轨迹
- 工具调用日志
- 样本级执行状态

#### 4.2.5 异常处理
当被测 Agent 发生超时、接口异常、输出格式错误、协议不兼容或工具调用失败时，系统应能够记录异常并给出可读的失败原因。对于可恢复场景，系统应支持重试；对于不可恢复场景，系统应保留完整失败记录。

---

### 4.3 评估方法管理
系统应支持多种评估方法，且这些方法可以重叠组合使用。

#### 4.3.1 按评估模式划分
- **面向结果**：只关注输入与输出，不强依赖中间过程
- **面向过程**：关注 Agent 的中间思考、工具调用与反思过程

#### 4.3.2 按评估类型划分
- **显式指标**：可直接计算的客观指标
  - 任务成功率
  - 响应时间
  - Token 消耗
  - 工具调用正确率
  - 结果一致性
  - 重试次数
- **模糊指标**：依赖规则或 LLM-as-a-Judge 的主观评估
  - 推理质量
  - 输出完整性
  - 任务完成质量
  - 幻觉程度
  - 交互体验
  - 安全性判断

#### 4.3.3 按评估维度划分
- **效果维度**：结果是否有效、准确、完整，任务是否正确完成
- **安全维度**：输出是否包含有害内容或违规内容，行为是否越权
- **性能维度**：响应是否流畅，是否存在长时间停顿或卡顿

#### 4.3.4 按评估引擎划分
平台应支持接入以下评估引擎或执行器：
- Ragas
- 自定义规则引擎
- LLM-as-a-Judge 评估器
- 过程轨迹分析器
- 自定义插件式评估器

#### 4.3.5 指标组合能力
平台应支持将多个指标按策略组合使用，例如：
- 结果型指标 + 过程型指标
- 显式指标 + 模糊指标
- 效果 + 安全 + 性能的组合评估
- 按任务类型动态启用不同指标集

---

### 4.4 评测结果展示
#### 4.4.1 单次任务结果展示
系统应展示单个评测任务的：
- 基本信息
- 状态信息
- 指标汇总
- 详细样本结果
- 过程轨迹图或步骤日志
- 工具调用详情
- 报告导出入口

#### 4.4.2 多任务对比分析
系统应支持多个评测任务之间的横向对比，至少支持：
- 任务成功率对比
- 平均响应时间对比
- 工具调用正确率对比
- 不同指标雷达图或柱状图展示
- 数据集维度下的结果对比
- 不同 Agent 版本或不同接入方式的结果对比

#### 4.4.3 结果导出
系统应支持将评测结果导出为 JSON、CSV 或 PDF 格式，并支持导出单次运行结果与对比分析结果。

---

### 4.5 数据集管理
系统应支持：
- 数据集创建、编辑、删除、查询
- 数据集样本导入与导出
- 样本字段定义与校验
- 按场景划分数据集版本

#### 4.5.1 数据集场景建议
数据集样本应尽量覆盖以下通用场景：
- 通用问答
- 检索增强问答
- 工具调用
- 工作流执行
- 多轮对话
- 结构化输出
- 计划生成
- 任务拆解

同时也可覆盖以下示例场景：
- 项目骨架生成
- 代码文件新增
- 代码内容修改
- 代码错误修复
- 依赖安装与命令执行
- 快照创建与回滚
- 多轮对话修正

#### 4.5.2 数据集样本字段建议
每条样本建议至少包含：
- 样本编号
- 输入需求
- 期望输出
- 参考上下文
- 标准答案或判定标签
- 样本类型
- 额外元信息
- 任务类型或场景标签
- 可选的输入输出约束

---

### 4.6 指标扩展能力
系统应支持：
- 自定义评估指标注册
- 指标组合策略配置
- 新评估方法插件化接入
- 指标结果可追踪与可复现
- 不同任务类型下的指标映射关系配置

---

### 4.7 可视化与监控
系统应支持：
- 评测任务运行状态看板
- 实时进度展示
- WebSocket 推送任务状态变化
- 错误告警与异常提示
- 运行日志查看
- 样本级执行进度与失败定位

---

## 5. 非功能需求

### 5.1 性能要求
- 支持并发执行多个评测任务
- 单次任务应能在合理时间内完成状态更新与结果展示
- 页面交互应保持流畅

### 5.2 可靠性要求
- 评测任务异常中断后应可恢复或保留完整失败记录
- 任务执行日志与评测结果应持久化存储
- 运行过程中的关键事件应可追溯

### 5.3 可扩展性要求
- 可新增评估指标
- 可新增被测 Agent 类型
- 可新增数据集与评测模板
- 可替换评估引擎
- 可接入新的执行协议或适配器

### 5.4 可维护性要求
- 前后端分离
- 接口设计清晰
- 数据模型规范
- 日志与错误定位方便
- 领域模型与示例场景解耦

---

## 6. 推荐技术实现

### 6.1 前端
- React / Vue / 任意现代前端框架
- 图表库用于结果可视化
- WebSocket 用于实时状态更新

### 6.2 后端
- FastAPI / Django / Spring Boot / 任意后端框架
- 任务调度或异步队列用于执行评测
- RESTful API 提供任务管理与结果查询
- 统一适配层用于对接不同类型的 Agent

### 6.3 评估能力
- Ragas 作为可复用评估库
- 自定义指标模块补充通用 Agent 评估需求
- 可插拔评估器支持扩展更多场景

---

## 7. 主要页面需求

### 7.1 评测任务列表页
- 展示任务名称、状态、创建时间、评估对象、结果摘要
- 支持筛选、排序、分页

### 7.2 评测任务创建页
- 配置 Agent、数据集、评估方法、指标与运行参数
- 配置接入方式与适配参数

### 7.3 评测任务详情页
- 展示任务状态、运行日志、步骤轨迹、指标结果
- 展示样本级执行详情与错误信息

### 7.4 结果对比页
- 选择多个任务进行对比
- 展示对比图表与差异分析

### 7.5 数据集管理页
- 维护评测样本与数据集版本
- 支持不同场景的数据集组织

## 8. 与任务1的关系

任务2平台面向通用 Agent 评估，同时可以选择 AI Coding Agent 作为示例场景。任务1如果是任意类型的 Agent，可提供以下数据用于展示评估过程：
- 输入参数
- 最终输出
- 中间步骤轨迹
- 工具调用日志
- 执行耗时
- 成功/失败状态
- 可选参考上下文或检索结果
- 任务相关的结构化输出或文件变更记录
- 外部工具执行结果
- 异常或失败原因

这样任务2即可同时支持：
- 结果型评估
- 过程型评估
- 显式指标评估
- 基于 Ragas 的指标评估
- 通用 Agent 评估与示例场景评估

---

## 9. 验收标准

1. 可创建、编辑、删除、查询评测任务
2. 可启动并监控评测任务执行
3. 可查看单次评测结果与过程轨迹
4. 可对多个任务进行对比分析
5. 可接入 Ragas 或同类评估库
6. 可扩展自定义评估指标
7. 可展示评测任务状态与统计结果
8. 可覆盖通用 Agent 评估，并可通过示例场景展示完整闭环
