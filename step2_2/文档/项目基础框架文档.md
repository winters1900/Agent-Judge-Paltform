# 项目基础框架文档

## 1. 项目概述

### 1.1 项目名称
Agent 应用评估平台

### 1.2 项目定位
本项目是一个面向 Agent 应用的评估平台，用于对 Agent 的任务完成情况、过程表现、工具调用情况、性能表现和安全表现进行统一评测、记录与分析。

### 1.3 项目目标
- 提供统一的 Agent 评测入口
- 支持评测任务的创建、配置、执行与管理
- 支持评测过程的实时监控与状态展示
- 支持评测结果、指标与轨迹的可视化分析
- 支持单任务分析与多任务对比
- 支持后续扩展自定义指标与评估方法

### 1.4 项目范围
本项目重点实现以下能力：
- 评测任务管理
- 数据集管理
- 评估方法管理
- 指标管理
- 评测执行与调度
- 过程轨迹记录
- 工具调用日志记录
- 结果统计与展示
- 多任务对比分析

## 2. 技术框架选型

### 2.1 前端技术栈
- **React + TypeScript + Ant Design**：构建管理后台页面
- **ECharts**：用于结果统计图表与对比分析展示

### 2.2 后端技术栈
- **FastAPI**：提供 REST API 与 WebSocket 服务
- **SQLAlchemy**：数据库 ORM 映射
- **MySQL 8.x**：存储任务、运行、样本、指标和报告数据
- **Redis**：缓存、状态同步和任务队列辅助存储
- **Celery**：异步任务调度与评测执行
- **Ragas**：用于部分评估指标计算，尤其是与检索、上下文质量、答案质量相关的评估

### 2.3 实时通信
- **WebSocket**：用于评测运行状态、进度和日志的实时推送

### 2.4 部署与运行建议
- 前后端分离部署
- 后端服务与任务 worker 分离
- MySQL 与 Redis 独立部署
- 评测执行任务通过 Celery worker 异步处理

## 3. 系统架构说明

### 3.1 架构分层
系统整体建议分为以下层次：

1. **表现层**
   - 前端页面
   - WebSocket 状态展示
   - 结果图表和任务管理界面

2. **接口层**
   - 评测任务 API
   - 数据集 API
   - 指标 API
   - 运行结果 API

3. **业务层**
   - 任务创建与配置
   - 任务调度与执行
   - 指标计算
   - 结果汇总与分析

4. **基础设施层**
   - MySQL
   - Redis
   - Celery worker
   - 文件存储

### 3.2 核心数据流
1. 用户在前端创建评测任务
2. 后端保存任务配置
3. 任务进入待执行队列
4. Celery worker 拉取任务并执行评测
5. 执行过程中写入轨迹、日志和中间结果
6. 系统计算指标并生成汇总结果
7. 前端实时查看任务状态、过程与结果

## 6. 典型业务流程

### 6.1 创建并执行评测任务
1. 用户创建评测任务
2. 选择被测 Agent、数据集和指标
3. 保存任务配置
4. 点击启动评测
5. 系统将任务投递到 Celery 异步执行队列
6. Worker 执行评测并记录过程
7. 系统生成结果并更新状态
8. 前端展示评测报告

### 6.2 多任务对比分析
1. 用户选择多个已完成任务
2. 系统拉取各任务指标结果
3. 按相同指标进行聚合对比
4. 展示对比图表和差异分析

## 7. 与任务1的关联
本项目的任务2评测平台可直接对接任务1的 Agent 应用。任务1如果提供统一的执行入口、过程轨迹和工具调用日志，那么任务2即可对其进行：
- 面向结果评估
- 面向过程评估
- 显式指标评估
- 模糊指标评估
- 多任务对比分析
