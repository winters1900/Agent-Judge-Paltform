# 迭代一评估报告

## 1. 评估指标选取原因

本次评估围绕外卖 Agent 任务轨迹，选择了 4 类互补指标，覆盖“动作正确性—规划合理性—任务完成度”的完整链路：

1. **工具调用准确度（tool_call_accuracy）**
   - 评估 Agent 工具调用是否与期望步骤一致，强调**顺序一致**与**参数精确匹配**。
   - 适用于定位“工具选错/参数错/顺序错”的关键问题。

2. **工具调用 F1（tool_call_f1）**
   - 评估工具调用集合层面的覆盖程度（不强制顺序）。
   - 用于衡量“该调用的是否都调用到了、乱调用是否多”。

3. **规划合理性（plan_quality）**
   - 从轨迹结构层面评估冗余、缺失、步骤长度偏差等规划问题。
   - 用于反映是否存在“重复调用、死循环、逻辑断层”。

4. **任务完成度（task_completion，LLM 判别）**
   - 直接评估最终回复是否满足用户诉求与期望结果。
   - 通过 LLM 输出 `completed` + `missing`（缺失要素）实现可解释性判断。

上述指标互补：
- `tool_call_accuracy` 极严格（顺序+参数），适合“硬正确性”评估；
- `tool_call_f1` 偏召回视角；
- `plan_quality` 评估轨迹结构；
- `task_completion` 面向最终用户体验。

---

## 2. 指标实现过程（核心逻辑）

### 2.1 工具调用准确度
- 若预测/参考均为空，记 1.0；任一为空则 0.0。
- 序列对齐（默认 strict_order=True）：工具调用序列完全一致才给 `aligned=1`。
- 对齐后逐步比较参数：只检查参考参数中出现的 key，值完全相等计分。
- 评分公式：
  \[
  score = avg\_arg\_score \times coverage \times aligned
  \]

### 2.2 工具调用 F1
- 将每步工具调用（工具名 + 参数）当作集合元素。
- 计算 TP/FP/FN，输出 F1。
- 不强制顺序，体现“覆盖率”。

### 2.3 规划合理性
- 统计重复调用惩罚、缺失工具惩罚、步骤长度差异。
- 重复超过阈值扣分；缺失期望工具扣分；长度差距扣分。

### 2.4 任务完成度（LLM 判别 + 缺失要素）
- 使用 LLM 对 `user_query + ground_truth + final_answer` 进行判别。
- 要求 LLM 输出 JSON：
  ```json
  {"completed": 0或1, "reason": "...", "missing": ["要素1", "要素2"]}
  ```
- 规则增强：**只要 `missing` 非空，强制 `completed=0`**，避免“缺失但仍完成”的情况。
- 若 LLM 调用失败，回退到规则版关键词匹配。

## 3. 评估结果分析（最新一次运行）

从 `迭代1/reports/eval_report.md` 汇总结果：

| 指标 | 均值 |
| --- | --- |
| tool_call_accuracy | 0.4537 |
| tool_call_f1 | 0.6987 |
| task_completion | 1.0000 |
| plan_quality | 0.9500 |

### 3.1 主要观察

1. **工具调用准确度偏低（0.4537）**
   - 说明严格顺序+参数对齐时，错误较多。
   - 常见原因：工具顺序不一致、参数值错误、遗漏步骤。

2. **工具调用 F1 较高（0.6987）**
   - 表明大部分核心调用“被覆盖”，但顺序和参数仍有问题。
   - 说明模型“知道该用什么工具”，但执行细节不足。

3. **规划合理性较高（0.95）**
   - 轨迹结构整体较干净，冗余与长度偏差较少。
   - 说明规划结构合理，但执行精度不足。

4. **任务完成度过高（1.0）需谨慎解读**
   - LLM 判别倾向于“给正例”，即使有缺失要素。
   - 虽已加 `missing` 强制规则，但仍存在 LLM 对“缺失要素”的判断偏宽松问题。

---

## 4. 改进建议

1. **对任务完成度增加硬约束**
   - 在代码侧加入关键要素规则（如必须包含订单号/支付方式）。
   - 与 LLM 判别做交叉验证，避免“全部 1.0”。

2. **引入日志与抽样评估**
   - 解决 API 限流，必要时增加调用间隔或抽样评估。

3. **加强参数一致性判断**
   - 对 `input` 进行更鲁棒的比较（如忽略字段顺序、允许数值近似）。

---

## 5. 结论

目前指标实现已覆盖工具调用正确性、规划合理性和任务完成度三个层面。整体表现显示：
- Agent 规划结构合理，工具调用覆盖率较高；
- 但严格准确度仍有明显不足；
- LLM 判别的任务完成度需进一步加强约束，避免结果偏乐观。

后续将通过规则+LLM 混合判别、降低限流影响、完善参数容错等方式提升评估可靠性。
